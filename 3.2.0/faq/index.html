2、事件​

OnEnter： 网赚界面打开时触发，根据自己需要注册使用。（如可在打开网赚界面时根据需要处理暂停游戏、禁掉可能的UI穿透触摸、隐藏Banner、Native广告等）

OnExit： 网赚界面关闭时触发，根据自己需要注册使用。（如可在关闭网赚界面时根据需要处理恢复游戏、恢复UI点击触摸等）

OnInvited：用户成功填入邀请码时触发，一般用于网赚/非网赚内容展示刷新。（如钻石/Robux资源条显示切换等）

3、属性​

IsAccessible：只读属性，用于获取网赚功能是否开启。（如false为网赚功能未开启，显示钻石资源条，true为网赚功能开启，显示Robux资源条）

4、接口​

EnableMMO：网赚功能开关。第一个参数是总开关，第二个为是否保持使用loading。【一般无需调用】

LogMMOUser：网赚重要打点。传入是否网赚用户，必须在TKGSDK中获取是否网赚用户的回调里调用。（可参照下方示例代码）【必须调用】

Init：初始化方法，游戏初始化时调用。可在TKGSDK的Init方法后立即调用。只需执行一次。第一个参数传实现了IMMOAgent的对象，第二个参数传Init成功回调。（调用Init方法默认会加载网赚内部的Loading界面，请参考自定义UI部分替换为自己游戏对应图片资源）。【必须调用】

StartGame：开始游戏方法，每关开始游戏前调用。必须保证在Init回调执行完之后再调用。只需传一个开始游戏成功回调。回调一般可传空，如有必须网赚引导结束后才能执行的内容，那需要放到开始游戏成功回调里。调用StartGame方法时会关掉Loading界面的展示。【必须调用】

SetMMOUI：设置网赚UI的方法。用于设置MMOBtns和MMOLevelBar的显示隐藏及对应数值，参数可以为空，可分开单独设置2个UI，内部会自动初始化，需要使用对应UI的地方调用就行，StartGame前后均可。【必须调用】

EndGame：结束游戏方法，每次通关成功前调用，失败不调用。第一个参数是结束回调，所有的游戏结算逻辑在回调里执行，回调参数为true时，可正常展示插屏和Native广告，false时不展示。第二个参数为是否完美通关，有完美通关概念的游戏才需要传。【必须调用】

WatchRVSuccessfully：每次看激励视频广告成功后都需要调用这个方法，网赚内部统计用。【必须调用】

GetRewardDiamondOfLevel：结算页面用来获取当前结算可获得基础钻石数的方法，传入玩家当前钻石数和关卡数。（注意：前3关不能有钻石翻倍的激励视频按钮）【必须调用】

RewardGet：结算页面点击激励翻倍按钮或者NoThanks后需要调用，点激励翻倍传true，NoThanks传false。【必须调用】

DeleteAccount：删除当前用户服务器数据，方便测试用，可以在Debug模式里加上这个功能。【按需调用】

参考代码 SDKManagerDemo.cs：SDKManagerDemo.cs

五、注意事项​

网赚模块只用到一种货币，钻石或Robux币。原则：非网赚用户看不到任何Robux相关内容。
游戏内根据 IsAccessible属性 来切换所有涉及该货币的显示UI。该属性为true则货币为Robux币，否则为钻石。（全文中提到的钻石/Robux币都表示这一种网赚内用到的货币）

  1. 接入网赚模块需要在自己的游戏数据中加入一个钻石属性。并根据美术效果图在游戏UI内体现钻石资源条（一般在游戏右上角和结算页面，也供新增钻石后有飞钻石到资源条的动画用）。

  2. 前三关结算不能展示倍数滚动条和钻石翻倍激励视频按钮。会导致前面因为翻倍加钻石巨多后面加钻石巨少从而不能达到激励用户的作用。正常失败可不奖励钻石数量，若奖励则前三关失败默认获得0钻石，后面失败都获得基础钻石的30%。基础钻石为通过 GetRewardDiamondOfLevel 接口获得的钻石数量。

  3. 网赚开关的控制，是根据TKGSDK中 SetFunctionSwitchListener 对 Function_wangz 功能的监听，返回true，则视为网赚用户，否则为非网赚用户。该监听为异步回调，需要一定时间。需要在网赚Init调用的同时立即调用。（请注意该监听需要在TGKSDK的Init方法之后调用，以保证正常监听）。

  4. 打开网赚页面的时候注意要禁掉游戏中可能会穿透UI执行的操作，关闭网赚页面后再恢复。

  5. 请注意打开网赚页面时，不要被Banner或Native广告遮挡。

六、常见问题​

1. 货币系统​

游戏内必须要有货币系统，网赚用户需展示Rubox货币，非网赚用户可不展示，如下所示：

2. 飞金币特效​

若实现起来较困难或费时，可使用网赚模块自带的 ResBar.prefab，详细使用方法参见本文 第三章-通用控件

3. 网赚模块显示问题​

如果网赚用户正确的显示了网赚内容后，当前进程中或再次重启后又不显示网赚内容时，游戏端应在 SetFunctionSwitchListener 方法中存储数值，从而保证网赚用户能一直展示网赚内容。

4. 网赚模块里气泡奖励间隔不正确​

网赚模块中气泡奖励间隔是每三关一次，如显示不正确时，检查是否在每关开始时调用了 MMOModule.Instance.StartGame();

6. 广告遮挡问题​

问题如图：

// 出现native广告遮挡时，可以在网赚页面打开方法中移除native广告。

MMOModule.Instance.OnEnter += removeNative;

// 移除 native 广告

7. 卡在loading页面无法继续​

请确认每次加载游戏场景中有调用： MMOModule.Instance.StartGame();
